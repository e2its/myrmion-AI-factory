# Plantilla E: API Integration Test (`tests/api/{{feature}}.api.test.{ext}`)

> **Purpose:** Template for HTTP-level API integration tests that validate actual endpoints against contracts from `contracts/`.
> These tests complement unit tests (which mock dependencies) by testing the **full HTTP request-response cycle**.
>
> **When to generate:** Feature has API endpoints defined in `design.md` or `contracts/`.
> **Source of truth:** `contracts/` (OpenAPI/GraphQL/TypeScript schemas generated by ARCH).
> **Runner:** Stack-specific — supertest (Node.js), httpx/TestClient (Python/FastAPI), net/http/httptest (Go), MockMvc (Java/Spring).
> **Incremental-slicing note.** Under `slicing_strategy: incremental`, API tests you generate are scoped to the increment currently in `BUILDING`. `increment_plan.md § 1` lists each increment's `contract_surface` — test ONLY those operations in this increment's PR. Cross-increment integration belongs in the final increment's acceptance gate.

## Pattern: Node.js (Express/Fastify + Supertest)

```typescript
// tests/api/{{feature}}.api.test.ts
// Generated by Agent: IMPLEMENT | Feature: {{FEATURE_ID}}
// Contract: contracts/openapi/{{CONTRACT_SLUG}}/v1.yaml

import request from 'supertest';
import { app } from '{{APP_ENTRY}}'; // e.g., '../../src/app'

describe('{{FEATURE_NAME}} API', () => {
  // ─── Happy Path (from spec.feature scenarios) ───
  describe('POST /api/v1/{{resource}}', () => {
    it('should create resource with valid payload → 201', async () => {
      const payload = {
        // Fields from user_journey.md DataIn schema
        // or from contracts/ request body schema
      };

      const response = await request(app)
        .post('/api/v1/{{resource}}')
        .send(payload)
        .expect(201);

      // Validate response matches contracts/ response schema
      expect(response.body).toMatchObject({
        // Fields from contracts/ response schema or user_journey.md DataOut
      });
    });
  });

  describe('GET /api/v1/{{resource}}/:id', () => {
    it('should return resource by id → 200', async () => {
      const response = await request(app)
        .get('/api/v1/{{resource}}/{{TEST_ID}}')
        .expect(200);

      expect(response.body).toHaveProperty('id');
      // Validate against contracts/ response schema
    });

    it('should return 404 for non-existent resource', async () => {
      await request(app)
        .get('/api/v1/{{resource}}/non-existent-id')
        .expect(404);
    });
  });

  // ─── Edge Cases (from test_plan.md Section 2) ───
  describe('Error Handling', () => {
    it('should reject malformed JSON → 400', async () => {
      await request(app)
        .post('/api/v1/{{resource}}')
        .set('Content-Type', 'application/json')
        .send('{ invalid json')
        .expect(400);
    });

    it('should reject missing required fields → 400/422', async () => {
      await request(app)
        .post('/api/v1/{{resource}}')
        .send({}) // Empty payload
        .expect(400); // or 422 per api-standards.instructions.md
    });
  });

  // ─── Security (from test_plan.md Section 2 + sec patterns) ───
  describe('Security', () => {
    it('should reject unauthenticated request → 401', async () => {
      await request(app)
        .get('/api/v1/{{resource}}')
        // No auth header
        .expect(401);
    });

    it('should sanitize SQL injection attempt → 400', async () => {
      await request(app)
        .post('/api/v1/{{resource}}')
        .send({ field: "' OR 1=1 --" })
        .expect(400);
    });
  });
});
```

## Pattern: Python (FastAPI + httpx/TestClient)

```python
# tests/api/test_{{feature}}_api.py
# Generated by Agent: IMPLEMENT | Feature: {{FEATURE_ID}}
# Contract: contracts/openapi/{{CONTRACT_SLUG}}/v1.yaml

import pytest
from httpx import AsyncClient  # or: from fastapi.testclient import TestClient
from {{APP_MODULE}} import app  # e.g., src.main

@pytest.mark.asyncio
class TestFeatureAPI:
    """API integration tests for {{FEATURE_NAME}}."""

    # ─── Happy Path ───
    async def test_create_resource_valid_payload_201(self, client: AsyncClient):
        payload = {
            # Fields from user_journey.md DataIn schema
        }
        response = await client.post("/api/v1/{{resource}}", json=payload)
        assert response.status_code == 201
        data = response.json()
        # Validate against contracts/ response schema
        assert "id" in data

    async def test_get_resource_by_id_200(self, client: AsyncClient):
        response = await client.get("/api/v1/{{resource}}/{{TEST_ID}}")
        assert response.status_code == 200

    async def test_get_resource_not_found_404(self, client: AsyncClient):
        response = await client.get("/api/v1/{{resource}}/non-existent")
        assert response.status_code == 404

    # ─── Edge Cases ───
    async def test_malformed_json_400(self, client: AsyncClient):
        response = await client.post(
            "/api/v1/{{resource}}",
            content="{ invalid",
            headers={"Content-Type": "application/json"},
        )
        assert response.status_code in (400, 422)

    async def test_missing_required_fields_422(self, client: AsyncClient):
        response = await client.post("/api/v1/{{resource}}", json={})
        assert response.status_code in (400, 422)

    # ─── Security ───
    async def test_unauthenticated_401(self, client: AsyncClient):
        response = await client.get("/api/v1/{{resource}}")
        # No auth header
        assert response.status_code == 401

    async def test_sql_injection_sanitized_400(self, client: AsyncClient):
        response = await client.post(
            "/api/v1/{{resource}}",
            json={"field": "' OR 1=1 --"},
        )
        assert response.status_code == 400
```

## Generation Rules

1. **One test file per API resource** (e.g., `users.api.test.ts`, `auth.api.test.ts`).
2. **Import app instance** (NOT start a server) — framework test utilities handle this.
3. **Test data** from `user_journey.md` DataIn/DataOut schemas or `contracts/` request/response schemas.
4. **Happy paths** from `spec.feature` scenarios → mapped to HTTP verbs + status codes.
5. **Edge/error cases** from `test_plan.md` Section 2 (Edge, Security, Negative rows with HTTP status codes).
6. **Status codes** from `contracts/` OpenAPI responses or `.claude/rules/api-standards.instructions.md`.
7. **Auth headers** if feature requires authentication (detect from `design.md` security section).
8. **File location:** `tests/api/` directory (stack-agnostic; `tests/` root from `constitution.md`).
9. **Runner command:** `./scripts/test.sh api --apply` or direct runner (`npx jest tests/api/`, `pytest tests/api/`, `go test ./tests/api/`).

## Contract Validation (Optional — Advanced)

If `.claude/rules/contract-first-policy.instructions.md` exists and `contracts/openapi/` contains OpenAPI specs:
- Use schema validation library (ajv for Node.js, jsonschema for Python) to validate response bodies match OpenAPI response schemas.
- This ensures runtime responses stay in sync with declared contracts.

```typescript
// Example: OpenAPI schema validation (Node.js)
import Ajv from 'ajv';
import * as fs from 'fs';
import * as path from 'path';
import yaml from 'js-yaml';

const openapiSchema = yaml.load(
  fs.readFileSync(
    path.join(__dirname, '..', '..', 'contracts', 'openapi', '{{CONTRACT_SLUG}}', 'v1.yaml'),
    'utf8',
  ),
) as Record<string, any>;

const ajv = new Ajv();
const validate = ajv.compile(openapiSchema.paths['/api/v1/{{resource}}'].post.responses['201'].content['application/json'].schema);

it('response matches OpenAPI schema', async () => {

  expect(validate(response.body)).toBe(true);
});
```
