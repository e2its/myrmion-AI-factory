## Reliability-scenario smoke blocks (integration scope addendum)

For scope=`integration` (emphasis on third-party adapters), these additional smoke blocks are MANDATORY in addition to the happy-path blocks above:

```
## SMOKE-REL-IDEMP — Idempotency replay
Preconditions: { caller has sent SMOKE-1 successfully }
Action: Repeat SMOKE-1 with the SAME idempotency_key.
Expected: Second response returns cached / idempotent result; dedupe store hit; NO duplicate downstream side-effect (DB row count unchanged, queue not re-published, external system not re-called).
Verdict: PASS | FAIL | BLOCKED

## SMOKE-REL-RETRY — Retry/backoff honoured
Preconditions: { toxiproxy / chaos tool configured OR staging fault-injection enabled }
Action: Force downstream to fail (503) for the first N attempts, then succeed.
Expected: Caller sees success; retry_count metric == N; latency within sum(exponential backoff + jitter); no tight-loop retry.
Verdict: PASS | FAIL | BLOCKED

## SMOKE-REL-DLQ — Dead-letter handling
Preconditions: { max_retries configured; DLQ topic exists }
Action: Send a payload that will fail deterministically past max_retries.
Expected: Message lands in DLQ with full context (payload + retry history + last error + timestamp); main queue unblocked; replay tooling can re-enqueue.
Verdict: PASS | FAIL | BLOCKED

## SMOKE-REL-SHUTDOWN — Graceful shutdown
Preconditions: { long-running request in-flight }
Action: Send SIGTERM to the service mid-request.
Expected: In-flight request completes within drain window; health endpoint goes unhealthy before new requests arrive; process exits 0 within terminationGracePeriodSeconds.
Verdict: PASS | FAIL | BLOCKED
```

Happy-path-only smoke (without REL-* blocks) is accepted for scope=`backend-only` that is NOT integration-flavoured (e.g. pure-internal API with no retry contract). When in doubt: include all four REL-* smokes — the cost is 20 minutes of dev smoke, the benefit is catching production-breaking reliability regressions on day one.
